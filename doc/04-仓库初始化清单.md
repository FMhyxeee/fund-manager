## 当前初始化 / 首次运行顺序

如果在新环境首次拉起项目，推荐顺序如下：

1. 安装依赖

```bash
uv sync --extra dev --extra data
```

2. 执行迁移

```bash
uv run alembic upgrade head
```

3. 选择性导入基金基础数据

```bash
uv run python scripts/import_all_funds.py
```

4. 导入持仓或交易

- 使用 API 导入
- 或直接调用导入模块 / 脚本

5. 运行 daily / weekly / monthly job 验证链路

```bash
uv run fund-manager-scheduler daily --portfolio-id 1
uv run fund-manager-scheduler weekly --portfolio-id 1
uv run fund-manager-scheduler monthly --portfolio-id 1
```

## 经验总结

bootstrap 阶段的几个关键决定已经被证明是正确的：

1. 先做确定性账本，再做 Agent。
2. 先做导入和净值同步，再做报告和策略讨论。
3. 脚本入口保留，但业务写逻辑要收敛到服务层。
4. 规划文档必须持续对齐真实实现，否则很快失真。
