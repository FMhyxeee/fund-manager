## 文档定位

本文档描述当前仓库的实际技术结构、运行方式和主要接口。

它回答的问题：

- 当前代码是怎么分层的

如果本文档和代码冲突，以代码为准；但文档应尽量反映当前真实实现，而不是历史设想。

## 当前技术定位

`fund-manager` 当前应被理解为：

- 一个个人、自托管的投资领域内核
- 一个以确定性会计、确定性决策和 append-only 留痕为核心的系统
- 一个向外部 runtime、UI 和自动化提供稳定 typed contract 的领域服务

AI 不是系统中心。

当前仓库里的 AI 相关代码只承担两类角色：

- 结构化 artifact 的 typed contract
- 最小可测试的 in-repo manual runtime adapter

生产级 model routing、provider 选择、外部搜索和渠道交互，应该属于 `OpenClaw` 或其他外部 orchestration runtime。

## 五类核心对象

当前仓库已经按下面的对象模型在收敛：

### 1. Canonical Facts

典型对象：

- `portfolio`
- `fund_master`
- `transaction`
- `position_lot`
- `nav_snapshot`
- `portfolio_snapshot`

主要模块：

- `storage/models.py`
- `storage/repo/*`
- `core/domain/*`
- `core/services/portfolio_service.py`
- `core/services/analytics_service.py`

### 2. Deterministic Decisions

典型对象：

- `decision_run`
- `decision_feedback`
- `decision_transaction_link`
- policy evaluation result

主要模块：

- `core/services/policy_service.py`
- `core/services/decision_service.py`
- `core/services/decision_feedback_service.py`
- `core/services/decision_reconciliation_service.py`

### 3. Research Signals

典型对象：

- watchlist candidates
- style leaders
- candidate fit

主要模块：

- `core/watchlist/service.py`

### 4. Fact Packs

作用：

- 作为 AI workflow 的 deterministic 输入
- 明确区分 workflow input 和 canonical storage model

当前模块：

- `core/fact_packs.py`

当前包含：

- `ReviewPositionFact`
- `WeeklyReviewFacts`
- `StrategyDebateFacts`

### 5. AI Artifacts

作用：

- 作为 AI narrative 的 typed 输出
- 与 canonical facts / deterministic decisions 分离

当前模块：

- `core/ai_artifacts.py`

当前包含：

- `ReviewAgentOutput`
- `StrategyAction`
- `StrategyProposalOutput`
- `ChallengerOutput`
- `JudgeOutput`

## Runtime 结构

当前 `agents/runtime` 的职责已经收缩为桥接层：

### `contracts.py`

定义 runtime-facing protocol：

- `ReviewAgent`
- `StrategyAgent`
- `ChallengerAgent`
- `JudgeAgent`

### `shared.py`

定义共享 helper：

- `PromptDefinition`
- `load_prompt_definition`
- `format_money`
- `format_ratio_as_percent`

### `*_agent.py`

仅保留 manual runtime 实现，主要用于：

- 本地最小闭环
- 测试
- typed contract 验证

这些模块不再拥有：

- canonical DTO
- persisted artifact schema
- 隐式领域真相

## 主要服务

### `AnalyticsService`

负责：

- 位置和组合层指标
- 日 / 周 / 月 / 区间收益
- 最大回撤

### `PortfolioService`

负责：

- 组装组合快照
- 汇总当前有效 lot
- 读取最新 NAV
- 生成 `PortfolioSnapshotDTO`

### `FundDataSyncService`

负责：

- 刷新当前持仓基金资料
- 增量写入 `nav_snapshot`
- 返回结构化同步摘要

### `DecisionService`

负责：

- 读取 active policy
- 判断 band breach
- 生成 deterministic action suggestion
- 形成 `decision_run`

### `DecisionFeedbackService`

负责：

- 记录人工 feedback
- 维持 append-only execution audit

### `FundWatchlistService`

负责：

- watchlist candidates
- style leaders
- candidate fit

## Workflow

### `daily_snapshot`

1. 刷新持仓基金资料与 NAV
2. 组装并保存 `portfolio_snapshot`

### `daily_decision`

1. 读取快照与 active policy
2. 生成 deterministic `decision_run`
3. 需要时供外部 runtime 做解释，但不允许替代结果

### `weekly_review`

1. workflow coordinator 组装 `WeeklyReviewFacts`
2. runtime 生成 `ReviewAgentOutput`
3. exporter 渲染 markdown
4. 持久化 `review_report`

### `monthly_strategy_debate`

1. workflow coordinator 组装 `StrategyDebateFacts`
2. Strategy / Challenger / Judge 依次运行
3. 保存 `strategy_proposal` 和 debate log

## Transport

### API

角色：

- canonical read/write 的主入口
- workflow trigger 的主入口

### CLI

角色：

- 本机 automation
- 运维和排障

主要入口：

- `fund-manager-admin`
- `fund-manager-scheduler`

### MCP

角色：

- read-mostly 查询入口
- 外部 agent client 的轻量读取界面

### Internal typed tools

角色：

- `fund-manager` 内部 workflow 调用面
- 尽量最小、typed、可审计

## 当前工程约束

1. 确定性 truth 优先于 AI narrative。
2. `core/fact_packs.py` 和 `core/ai_artifacts.py` 是当前 AI workflow 的稳定 contract。
3. `agents/runtime` 是 adapter layer，不是领域真相层。
4. 脚本不应复制 repo / service 写逻辑。
5. 缺失 NAV 必须显式暴露，不能默认补零。
6. 历史快照、决策、反馈、报告和策略提案都应 append-only。
