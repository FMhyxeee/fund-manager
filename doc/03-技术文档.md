# 基金投资管理器技术文档（Technical Specification）

## 文档定位

本文档记录当前仓库的实际技术结构、运行方式和接口边界。

它回答的问题：

- 仓库怎么组织
- 现有哪些核心模型和服务
- API / Scheduler / MCP 分别提供什么
- 如何运行和测试

## 技术栈

- Python 3.12+
- FastAPI
- SQLAlchemy 2.x
- Alembic
- Jinja2
- AKShare
- pytest / mypy / ruff

可选 extra：

- `data`: `akshare`、`pandas`
- `mcp`: `mcp`
- `dev`: `pytest`、`mypy`、`ruff`

## 核心模型

主要持久化模型位于 `storage/models.py`：

- `FundMaster`
- `Portfolio`
- `TransactionRecord`
- `PositionLot`
- `NavSnapshot`
- `PortfolioSnapshot`
- `ReviewReport`
- `StrategyProposal`
- `AgentDebateLog`
- `SystemEventLog`

设计约束：

- 历史快照和日志为 append-only
- `position_lot` 支持 bootstrap 批次和后续重建批次
- 缺失净值时允许组装不完整快照，但不允许保存完整值快照

## 核心服务

### `AnalyticsService`

负责：

- 位置和组合层的收益指标
- 日 / 周 / 月 / 区间收益
- 最大回撤

### `PortfolioService`

负责：

- 汇总当前有效持仓
- 读取最新净值
- 组装组合快照 DTO
- 保存 `portfolio_snapshot`

### `FundDataSyncService`

负责：

- 找出某个组合当前有效持仓基金
- 刷新基金基础资料
- 增量写入 `nav_snapshot`
- 返回结构化同步摘要

## 数据接入

### CSV 导入

当前已实现：

- `import_holdings.py`
- `import_transactions.py`

都支持：

- 字段校验
- 数值标准化
- `dry_run`
- 明确的导入摘要

### AKShare 适配器

当前已实现：

- `search_fund()`
- `get_fund_profile()`
- `get_fund_nav_history()`

设计要求：

- 所有第三方字段映射都留在适配器层
- 对外返回内部 DTO
- 失败和空结果显式处理

## REST API

FastAPI 应用入口：

- `fund_manager.apps.api.main:app`

当前路由：

| Method | Path | 说明 |
| --- | --- | --- |
| `GET` | `/api/v1/health` | 健康检查 |
| `GET` | `/api/v1/dashboard` | Dashboard 页面 |
| `GET` | `/api/v1/portfolios` | 组合列表 |
| `GET` | `/api/v1/portfolios/{portfolio_id}/snapshot` | 组合快照 |
| `GET` | `/api/v1/funds/{fund_code}` | 基金详情 |
| `GET` | `/api/v1/reports` | 周报列表 |
| `POST` | `/api/v1/imports/holdings` | 导入持仓 |
| `POST` | `/api/v1/imports/transactions` | 导入交易 |
| `POST` | `/api/v1/workflows/weekly-review/run` | 触发周报 |

当前未作为 REST 暴露的能力：

- strategy debate workflow
- daily fund sync
- fund search
- fund NAV history 查询

## Scheduler

CLI 入口：

- `fund-manager-scheduler`
- `fund_manager.scheduler.cli:main`

支持频率：

- `daily`
- `weekly`
- `monthly`

当前注册 job：

- `daily_snapshot`
- `weekly_review`
- `monthly_strategy_debate`

其中：

- `daily_snapshot` 会先同步基金数据，再保存组合快照
- `weekly_review` 会持久化 `review_report`
- `monthly_strategy_debate` 会持久化 `strategy_proposal`

## MCP

MCP 为可选能力，安装 `mcp` extra 后可用。

服务入口：

- `fund-manager-mcp`
- `fund_manager.mcp.server:main`

当前工具：

- `portfolio_list`
- `portfolio_snapshot`
- `portfolio_positions`
- `portfolio_valuation_history`
- `portfolio_metrics`
- `fund_profile`
- `fund_nav_history`
- `simulate_model_portfolio`

## 脚本

当前仓库中的脚本用途：

- `scripts/import_all_funds.py`
  - 一次性批量导入基金主数据
- `scripts/import_all_nav_history.py`
  - 批量回填基金净值历史
- `scripts/portfolio_daily_report.py`
  - 通过 canonical 数据生成每日持仓报告


```bash
uv sync --extra mcp

```

## 开发约束

1. 权威数字必须来自确定性服务。
2. 导入和外部数据写入必须先标准化，再入库。
3. 新增 schema 必须带迁移和测试。
4. 脚本不能复制业务写逻辑，应复用服务层。
5. 对缺失 NAV 必须显式暴露，不允许默认补零。
